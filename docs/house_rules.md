# House Rules

**Target user:** Small business owners who are not technical. If a feature needs a manual, it is too complicated.

**Budget ceiling:** Hosting and services must stay under $20/month. Ask before adding anything paid.

**Never without asking:**
- Never touch payment or checkout code without explicit approval
- Never delete customer data — archive instead
- Never add a new external service or dependency without flagging it first

**Always:**
- Every user-facing flow must have a visible error state
- Plain English everywhere the customer looks — no technical jargon
- Mobile has to work; most customers order from their phone

**Tone:** Friendly and direct, like a helpful local shop — never corporate.
